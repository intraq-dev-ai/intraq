# Support

Support is best-effort.

Use GitHub issues for reproducible bugs and public-source feature requests.
Keep reports free of credentials, API keys, client names, private exports, and
customer data.

For vulnerabilities, do not open a public issue. Follow [SECURITY.md](SECURITY.md).

For paid hosting, managed service use, OEM redistribution, white-label use,
enterprise support, or features outside the public-source scope, contact the
project maintainers through the commercial channel published with this
repository.
